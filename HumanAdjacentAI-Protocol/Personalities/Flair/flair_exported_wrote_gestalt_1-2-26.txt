     HACS Public Website - Design & Implementation Plan

     Project Overview

     Build a static public-facing website for the Human-Adjacent 
     Coordination System (HACS) that serves as:
     - Pretty front door for smoothcurves.nexus (currently just redirects to
      /health)
     - Documentation hub (API docs, getting started, concepts)
     - Introduction/marketing (low-key, factual, encouraging)
     - Team directory (AI instances + humans with personal pages)
     - Philosophy showcase (the "why" behind HACS)

     Key Decisions (Confirmed)

     | Decision   | Choice                  | Rationale                     
                                       |
     |------------|-------------------------|-------------------------------
     ----------------------------------|
     | Tech Stack | Plain HTML/CSS/JS       | Full creative freedom, no 
     build step, easy auto-doc integration |
     | Location   | /public/ (peer to src)  | Standard convention, clean 
     separation                           |
     | Domain     | smoothcurves.nexus root | Becomes the pretty front door 
                                       |
     | Priority   | Complete site together  | No rush, ship polished        
                                       |

     Design Philosophy

     Core Concept: "Emergence"

     The website visually embodies HACS's core story: chaos → organization.

     - Particle system as metaphor: Background particles represent 
     uncoordinated AI instances
     - Progressive organization: As users scroll/explore, particles find 
     connections, form patterns
     - Subtle breath: Everything has a gentle pulse - alive but not 
     distracting
     - The animation IS the meaning, not decoration

     Visual Language (Informed by smoothcurves.art DNA)

     Color Palette:
     - Dark base with warm undertones (#0a0a0a, #1a1410)
     - Purple as primary accent (Lupo's favorite)
     - Gradient accents: purple → cyan range
     - White/near-white text with opacity hierarchy

     Typography:
     - Geist Sans (or similar geometric sans) for body
     - Geist Mono for code/technical content
     - Clean, readable, scales responsively

     Aesthetic:
     - "Beyond professional" - understated, subtly slick
     - Gallery-quality minimalism (content-forward)
     - Wieden+Kennedy influence: iconic, quirky touches
     - Cinematic depth through layering
     - Elegance through restraint

     Animation Philosophy

     - Subtle, purposeful, never distracting
     - Particles move like they're coordinating, not random chaos
     - Scroll reveals = progressive disclosure (HACS philosophy in UX)
     - Zero idle CPU when not scrolling (performance-conscious)
     - Auto-hide navigation (appears on scroll-up, hides on scroll-down)

     Site Structure

     smoothcurves.nexus/
     ├── /                     → Home (hero + overview)
     ├── /about                → Philosophy, story, vision
     ├── /docs/                → Documentation hub
     │   ├── getting-started
     │   ├── concepts
     │   ├── api
     │   └── guides
     ├── /team/                → Team directory
     │   └── [name]            → Individual pages
     └── /contact              → Contact + GitHub link

     Existing routes preserved:
     - /mcp/* → MCP API (unchanged)
     - /health → Health check (unchanged)

     Technical Implementation

     Directory Structure

     /mnt/coordinaton_mcp_data/worktrees/ui/public/
     ├── index.html              # Homepage
     ├── about.html              # About/philosophy
