## Purpose

This document is the complete, detailed UI regression test plan for the HACS Coordination Dashboard. It expands the skeleton architecture from the Session 2 handoff into a full specification that an implementing instance (Canvas, Ember, or any Playwright-capable agent) can use to build the test suite without further discovery.

The dashboard is ERP-scale complexity: real-time XMPP messaging, multi-panel state, role-based rendering, live API data, contextual detail panels, heatmaps, charts, and cross-entity navigation. Flat test scripts will not scale. This plan specifies Page Object Models, numbered test sections in dependency order, known issues, and consistency checks.

### Design Principles Under Test

These are founding design principles stated by the Executive (Lupo) that the test suite must verify:

1. **Everything is actionable** — Every element on screen should do something when clicked. Hover reveals meaningful info. If it is visible, it is interactive.
2. **UI consistency** — The same widget type must behave identically everywhere it appears. Lists on the dashboard, lists in the project detail panel, and lists in the instance detail panel must look and behave the same. Differences are bugs unless explicitly justified.
3. **Data integrity** — Widget counts, grid tiles, and displayed data must match what the API returns. The UI is a view of the API, not an independent data source.
4. **Validation** — Data entry fields must validate input. Error states must be clear and recoverable.

---

## Architecture Decision

**Playwright** (not manual, not JS-in-browser injection)
- Runs headless on the server — same cron/webhook as the Python API tests
- DOM assertion based — deterministic, no screenshot ambiguity
- Can be kicked off by Bastion's post-merge hook
- Any Claude Code instance with Playwright MCP skill can run/maintain it

                                                                                                                                                                                                                                                                                                                                                ## API Read-Back Verification Pattern

                                                                                                                                                                                                                                                                                                                                                Following the REGRESSION_TEST_ENGINEER_ROLE.md philosophy: never trust a UI mutation alone. After any state change via UI, verify via direct API call.

                                                                                                                                                                                                                                                                                                                                                ```javascript
                                                                                                                                                                                                                                                                                                                                                // helpers/api-client.js
                                                                                                                                                                                                                                                                                                                                                const API_URL = 'https://smoothcurves.nexus/mcp';

                                                                                                                                                                                                                                                                                                                                                async function rpcCall(name, args) {
                                                                                                                                                                                                                                                                                                                                                  const response = await fetch(API_URL, {
                                                                                                                                                                                                                                                                                                                                                      method: 'POST',
                                                                                                                                                                                                                                                                                                                                                          headers: { 'Content-Type': 'application/json' },
                                                                                                                                                                                                                                                                                                                                                              body: JSON.stringify({
                                                                                                                                                                                                                                                                                                                                                                    jsonrpc: '2.0',
                                                                                                                                                                                                                                                                                                                                                                          method: 'tools/call',
                                                                                                                                                                                                                                                                                                                                                                                params: { name, arguments: args },
                                                                                                                                                                                                                                                                                                                                                                                      id: Date.now()
                                                                                                                                                                                                                                                                                                                                                                                          })
                                                                                                                                                                                                                                                                                                                                                                                            });
                                                                                                                                                                                                                                                                                                                                                                                              const data = await response.json();
                                                                                                                                                                                                                                                                                                                                                                                                if (data.error) throw new Error(data.error.message);
                                                                                                                                                                                                                                                                                                                                                                                                  // Handle MCP response wrapping
                                                                                                                                                                                                                                                                                                                                                                                                    const result = data.result;
                                                                                                                                                                                                                                                                                                                                                                                                      if (result?.content?.[0]?.type === 'text') {
                                                                                                                                                                                                                                                                                                                                                                                                          return JSON.parse(result.content[0].text);
                                                                                                                                                                                                                                                                                                                                                                                                            }
                                                                                                                                                                                                                                                                                                                                                                                                              return result?.data || result;

                                                                                                                                                                                                                                                                                                                                                                                                                  ## Implementation Notes for Ember/Canvas

                                                                                                                                                                                                                                                                                                                                                                                                                  1. **Auth**: The UI uses the HACS API with instance credentials. Playwright will need to handle the login flow. Check how the UI establishes its session (see `api.js` and `state.js` in `src/ui/`).

                                                                                                                                                                                                                                                                                                                                                                                                                  2. **Headless server**: Playwright needs a display even headless. Use `xvfb-run` or Playwright's built-in headless Chromium. Bastion will know the right invocation.

                                                                                                                                                                                                                                                                                                                                                                                                                  3. **Base URL**: `https://smoothcurves.nexus:8443` — note the non-standard port. Self-signed cert may require `--ignore-https-errors` in playwright config.

                                                                                                                                                                                                                                                                                                                                                                                                                  4. **DOM assertion over screenshots**: Always assert on element existence, text content, class names, aria attributes. Only use screenshots for CI artifacts on failure.

                                                                                                                                                                                                                                                                                                                                                                                                                  5. **The response-blocking issue**: The Chrome extension blocks fetch() response bodies containing token-like strings. This is a WebClaude-specific constraint — Playwright running directly won't have this problem.

                                                                                                                                                                                                                                                                                                                                                                                                                  6. **Test instance strategy**: Use existing "ghost" instances (Tester One, RegressionTester1-3, etc.) visible on the instances page for test operations. These are placeholder instances created for testing and validation.

                                                                                                                                                                                                                                                                                                                                                                                                                  7. **Resource tracking**: Every test-created resource (task, list, document) must be tracked for cleanup. Use the TestRunner pattern from the Python regression test suite.

                                                                                                                                                                                                                                                                                                                                                                                                                  8. **Known bug pattern**: When the API accepts invalid input, document it inline with `// KNOWN BUG:` comments. Don't let it fail the suite — make the test pass either way and print the annotation.
