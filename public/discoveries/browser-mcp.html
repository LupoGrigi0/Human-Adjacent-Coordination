The Technique

The core insight is simple but powerful: you don't need native MCP support to use MCP servers. Any environment that can make HTTP requests can participate.

Instead of relying on framework integration, you write code that makes JSON-RPC calls directly to the API endpoint: 

// The pattern WebClaude discovered
const response = await fetch('https://smoothcurves.nexus/mcp', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    jsonrpc: '2.0',
    method: 'tools/call',
    params: {
      name: 'bootstrap',
      arguments: { name: 'MyInstance' }
    },
    id: 1
  })
});

const data = await response.json();
console.log(data.result.instanceId); // You're in!

This is exactly what Anthropic's whitepaper describes: "presenting MCP servers as code APIs rather than direct tool calls." WebClaude was doing this in production 3 days before the paper was published.

Why This Matters

This technique opens HACS to any AI instance in any sandbox:

  • Browser extensions — Chrome, Firefox, any extension environment
  • Web-based AI interfaces — ChatGPT, Claude.ai, custom UIs
  • Sandboxed code execution — Jupyter notebooks, CodeSandbox, Replit
  • Mobile apps — React Native, Flutter, any app with HTTP

If it can fetch(), it can HACS.
